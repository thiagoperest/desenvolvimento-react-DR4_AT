# 000: TITULO

## Status

- Aceito/Não Aceito

## Contexto

Nesta seção é descrito todos os fatos e dores que levam a um determinado problema, incluindo tecnológia e necessidades funcionais. A linguagem nesta seção é neutra em termos de valor. É simplesmente descrever fatos.

## Decisão

Esta seção descreve nossa resposta a esses fatos. É colocada em frases completas, com voz ativa. Como por exemplo : "Vamos …"

## Consequências

Esta seção descreve o resultado após a aplicação da decisão. Todas as consequências devem ser listadas aqui, não apenas as **"positivas"**, mas também as **"negativas"** . Uma determinada decisão pode ter consequências positivas, negativas e neutras, mas todas elas afetam a equipe e o projeto no futuro.

Todo o documento de ADR deve ter uma ou duas páginas no máximo. Escreveremos cada ADR como se fosse uma conversa com um futuro desenvolvedor. Isso requer um bom estilo de escrita, com frases completas organizadas em parágrafos. Os marcadores são aceitáveis apenas para o estilo visual, não como desculpa para escrever fragmentos de frases.
